# SVDH Local Development Environment - Copilot Instructions

## Project Overview

This is a Docker Compose setup for running SVDH (Secure Vereign Data Hub) services locally. It provides a complete development environment with all necessary infrastructure.

## Architecture

### Application Services

| Service | Image | Port | Database | Vault Mount | S3 |
|---------|-------|------|----------|-------------|-----|
| smimekeys-client | registry.vereign.io/svdh/smimekeys | 8081 | smimekeys_client | secret-smimekeys-client | No |
| policy | registry.vereign.io/svdh/policy | 8082 | policy | secret-policy | No |
| idagent | registry.vereign.io/svdh/idagent | 8083 | idagent | secret-idagent | Yes |
| mxengine | registry.vereign.io/svdh/mxengine | 8084, 1587 | mxengine | secret-mxengine | Yes |

### Infrastructure Services

| Service | Image | Ports | Purpose |
|---------|-------|-------|---------|
| postgres | postgres:15-alpine | 5432 | Database for all apps |
| vault | hashicorp/vault:1.19.0 | 8200 | Secrets management (KV-v2) |
| minio | minio/minio | 9000, 9001 | S3-compatible object storage |

### Service Dependencies

```
postgres ─────────────────┐
                          ├──► smimekeys-client
vault ──► vault-init ─────┤
                          ├──► policy
minio ──► minio-init ─────┤
                          ├──► idagent
                          └──► mxengine
```

## File Structure

```
.
├── docker-compose.yml      # Main orchestration file
├── .env                    # Environment variables (gitignored)

├── config/
│   └── vault/
│       └── vault.hcl       # Vault server configuration (file storage)
├── init/
│   └── postgres/
│       └── 01-create-databases.sql  # Creates DBs on first start
├── scripts/
│   ├── start.sh            # Main entry point - handles init & restart
│   ├── stop.sh             # Graceful shutdown
│   └── init-vault.sh       # Vault initialization (runs in container)

```

## Key Patterns

### Environment Variables

All services use environment variables for configuration. Pattern:
- Infrastructure config: `.env` file (docker-compose interpolation)
- App config: Defined inline in `docker-compose.yml` under each service's `environment` section
- Secrets: `VAULT_TOKEN` comes from `.env`, populated by `start.sh`

### Database Connection

All services use the same pattern:
```
DB_URI: postgres://${POSTGRES_USER}:${POSTGRES_PASSWORD}@postgres:5432/<database_name>
```

### Vault Configuration

Each service that uses Vault needs:
```yaml
VAULT_ADDRESS: "http://vault:8200"
VAULT_TOKEN: "${VAULT_TOKEN}"
VAULT_MOUNT_PATH: "secret-<service-name>"
```

### S3/MinIO Configuration

Services using S3:
```yaml
S3_HOSTNAME: "minio:9000"
S3_ACCESS_KEY: "${MINIO_ROOT_USER}"
S3_SECRET_KEY: "${MINIO_ROOT_PASSWORD}"
S3_SECURE: "false"  # No TLS locally
S3_BUCKET_NAME: "${S3_BUCKET_NAME}"
```

### Health Checks

All apps expose `/liveness` endpoint on their HTTP port:
```yaml
healthcheck:
  test: ["CMD", "wget", "-q", "--spider", "http://localhost:8080/liveness"]
  interval: 10s
  timeout: 5s
  retries: 5
  start_period: 30s
```

## Common Operations

### Adding a New Service

1. Add database in `init/postgres/01-create-databases.sql`:
   ```sql
   CREATE DATABASE new_service;
   GRANT ALL PRIVILEGES ON DATABASE new_service TO postgres;
   ```

2. Add Vault mount in `scripts/init-vault.sh`:
   ```bash
   vault secrets enable -address=http://vault:8200 -path=secret-new-service kv-v2 || echo "already exists"
   ```

3. Add service in `docker-compose.yml`:
   ```yaml
   new-service:
     image: registry.vereign.io/svdh/new-service:${NEW_SERVICE_VERSION:-latest}
     container_name: svdh-new-service
     depends_on:
       postgres:
         condition: service_healthy
       vault-init:
         condition: service_completed_successfully
     environment:
       DB_URI: "postgres://${POSTGRES_USER}:${POSTGRES_PASSWORD}@postgres:5432/new_service"
       VAULT_ADDRESS: "http://vault:8200"
       VAULT_TOKEN: "${VAULT_TOKEN}"
       VAULT_MOUNT_PATH: "secret-new-service"
       # ... other env vars
     ports:
       - "808X:8080"
     healthcheck:
       test: ["CMD", "wget", "-q", "--spider", "http://localhost:8080/liveness"]
       interval: 10s
       timeout: 5s
       retries: 5
       start_period: 30s
     networks:
       - svdh-network
   ```

4. Add version variable to `.env` and `.env.example`:
   ```
   NEW_SERVICE_VERSION=latest
   ```

### Adding Environment Variables to Existing Service

1. For non-sensitive: Add directly to `environment` section in `docker-compose.yml`
2. For sensitive: Add to `.env` file and reference as `${VAR_NAME}`

### Changing Vault Configuration

- Vault config: `config/vault/vault.hcl`
- Vault mounts: `scripts/init-vault.sh`
- Note: After changing mounts, you need to reset: `docker compose down -v && rm -rf secrets/`

### Changing Database Schema

PostgreSQL init scripts only run on first start. To re-run:
```bash
docker compose down -v  # Removes volumes
./scripts/start.sh      # Fresh start
```

## Configuration Reference

### smimekeys-client

Connects to external CA at `https://smimekeys-ca-dev.k8s.vereign-cdn.com`. Key settings:
- `CERT_CA_ROOT: "false"` - Not a root CA
- `CERT_CA_ADDR` - External CA address
- `LDAP_*` - LDAP configuration for certificate publishing

### policy

Simple policy service. Key settings:
- `DB_NOTIFIER_MAX_RETRY_ATTEMPTS` - Retry configuration

### idagent

Identity agent service. Key settings:
- `SERVICE_PUBLIC_ADDR` - Public callback URL
- `ISSUER_SERVICE_ADDR` / `VERIFIER_SERVICE_ADDR` - External Vereign services

### mxengine

Email processing engine. Key settings:
- `INBOUND_HANDLERS: "json,smtp"` - Enabled handlers
- `INBOUND_SMTP_*` - SMTP server configuration
- `SEALER: "false"` - Disabled for local (no outbound email)

## Troubleshooting Guide

### Vault Issues

**Vault sealed after restart:**
- Run `./scripts/start.sh` - it auto-unseals using stored keys

**Lost Vault keys:**
- Reset everything: `docker compose down -v && rm -rf secrets/`
- Run `./scripts/start.sh` for fresh initialization

**Cannot create mounts:**
- Check if Vault is unsealed: `docker exec svdh-vault vault status`
- Verify token: Check `secrets/vault-keys.json`

### Database Issues

**Service can't connect to database:**
- Check if postgres is healthy: `docker compose ps postgres`
- Verify database exists: `docker exec svdh-postgres psql -U postgres -l`

**Need to reset database:**
```bash
docker compose down
docker volume rm svdh-local_postgres_data
docker compose up -d
```

### Service Startup Issues

**Service in restart loop:**
1. Check logs: `docker compose logs <service>`
2. Common causes:
   - Database not ready (check depends_on)
   - Vault not unsealed
   - Missing environment variable

## Security Notes

- `secrets/` directory contains Vault unseal keys - never commit
- `.env` contains sensitive credentials - gitignored
- Vault runs without TLS (dev only)
- MinIO runs without TLS (dev only)
